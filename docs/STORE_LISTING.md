## 2. Краткое описание

**Play Market** (80 символов максимум):
```
Защищённый мессенджер с E2E-шифрованием. Чаты, голосовые, фото, уникальные ID.
```

**RuStore** (255 символов максимум):
```
iRomashka — приватный мессенджер с сквозным E2E-шифрованием на основе P-256 ECDH + AES-256-GCM. Голосовые сообщения, фото, видео, исчезающие сообщения, биометрия и биржа красивых номеров (УИН).
```

## 3. Полное описание

### Версия для Play Market и RuStore (до 4000 символов):

```
iRomashka — мессенджер нового поколения с настоящим сквозным шифрованием.

🔐 ПОЛНАЯ ПРИВАТНОСТЬ
Все сообщения зашифрованы по стандарту E2E (P-256 ECDH + HKDF-SHA256 + AES-256-GCM) — ни сервер, ни третьи лица не могут прочитать вашу переписку. Транспорт защищён протоколом ChaCha20. Приватный ключ хранится только на вашем устройстве, защищённый PIN-кодом через Argon2id.

💬 ВОЗМОЖНОСТИ
• Личные и групповые чаты
• Голосовые сообщения
• Отправка фото и видео
• Редактирование и удаление сообщений у всех
• Подтверждения прочтения (✓✓)
• Индикатор набора текста
• Исчезающие сообщения: автоудаление через 30 сек / 5 мин / 1 ч / 24 ч / 7 дней
• Биометрический вход (отпечаток пальца / Face ID)

🔑 ВОССТАНОВЛЕНИЕ ДОСТУПА
Мнемоническая фраза из 12 слов (BIP39) позволяет восстановить аккаунт и переписку при утере PIN.

🎯 УНИКАЛЬНЫЙ ID (УИН)
Ваш номер — ваш адрес в iRomashka. Биржа красивых УИН позволяет выбрать запоминающийся 6- или 7-значный номер.

📋 УПРАВЛЕНИЕ СЕССИЯМИ
Просматривайте и удаляйте активные устройства прямо из приложения.

🔒 БЕЗОПАСНОСТЬ
• Certificate Pinning — защита от MITM-атак
• PIN защищён Argon2id (m=19MB, t=2) — брутфорс практически невозможен
• Защита от скриншотов на экранах с чувствительными данными
• Буфер обмена очищается автоматически через 60 секунд
• Шифрование трафика запрещает перехват на уровне сети

📱 ТРЕБОВАНИЯ
Android 8.0 и выше. Для работы необходима регистрация по номеру телефона.
```

## 4. Data Safety (Google Play — раздел «Безопасность данных»)

### Собираемые данные:

| Тип данных | Собирается | Зашифровано | Передаётся третьим лицам |
|------------|-----------|-------------|--------------------------|
| Номер телефона | Да | Да (AES-256-GCM) | Нет |
| Сообщения | Да (только E2E ciphertext) | Да | Нет |
| Фото / видео | Да (только зашифрованные) | Да | Нет |
| UIN (пользовательский ID) | Да | — | Нет |
| FCM-токен (push) | Да | — | Firebase (Google) |
| IP-адрес | Логируется на сервере | — | Нет |

### Данные, которые **не собираются**:
- Контакты телефонной книги (используются локально только для поиска пользователей)
- Геолокация
- История браузера
- Финансовые данные (платежи через ЮKassa — внешний сервис)

### Ответы на вопросы формы Data Safety:

**Собирает ли приложение данные пользователей?** — Да  
**Передаются ли данные третьим лицам?** — Да (FCM токен → Google Firebase для push-уведомлений)  
**Можно ли запросить удаление данных?** — Да (через настройки → Удалить аккаунт)  
**Шифруются ли данные при передаче?** — Да (TLS 1.3 + E2E)  
**Соответствует ли приложение политике Play Families?** — Нет (приложение не для детей)

## 5. Разрешения (Permissions) — пояснения для магазинов

| Разрешение | Зачем нужно |
|------------|-------------|
| `INTERNET` | Соединение с сервером iRomashka |
| `POST_NOTIFICATIONS` | Уведомления о новых сообщениях |
| `RECORD_AUDIO` | Запись голосовых сообщений |
| `READ_MEDIA_IMAGES` | Выбор фото из галереи для отправки |
| `READ_CONTACTS` | Поиск контактов из телефонной книги среди пользователей iRomashka |
| `USE_BIOMETRIC` | Биометрический вход (отпечаток / Face ID) |
| `FOREGROUND_SERVICE` | Поддержание соединения для получения сообщений в реальном времени |
| `RECEIVE_BOOT_COMPLETED` | Автозапуск после перезагрузки для восстановления соединения |
| `VIBRATE` | Виброуведомления при новых сообщениях |
| `WAKE_LOCK` | Надёжная доставка сообщений в фоне |
| `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` | Стабильная работа мессенджера в фоне |

## 6. Политика конфиденциальности (Privacy Policy)

**Требуется разместить по URL**, например: `https://iromashka.ru/privacy`

### Текст политики:

```
Политика конфиденциальности iRomashka
Последнее обновление: май 2026

1. СОБИРАЕМЫЕ ДАННЫЕ
iRomashka собирает минимально необходимый набор данных:
- Номер телефона: хранится в зашифрованном виде (AES-256-GCM), используется только для регистрации и поиска
- Идентификатор пользователя (УИН): числовой ID, присваивается при регистрации
- Содержимое сообщений: хранится на серверах исключительно в зашифрованном виде (E2E); администрация сервиса не имеет технической возможности прочитать сообщения
- FCM-токен: передаётся Google Firebase исключительно для доставки push-уведомлений
- Технические логи (IP-адрес, временные метки): хранятся не более 90 дней

2. ИСПОЛЬЗОВАНИЕ ДАННЫХ
Данные используются для:
- Обеспечения работы мессенджера (доставка сообщений, аутентификация)
- Отправки push-уведомлений через Firebase Cloud Messaging
- Защиты от мошенничества и злоупотреблений (rate limiting)

3. ПЕРЕДАЧА ТРЕТЬИМ ЛИЦАМ
Мы не продаём и не передаём ваши данные третьим лицам, за исключением:
- Google Firebase: FCM-токен для push-уведомлений
- ЮKassa: данные платежей при покупке УИН через биржу (обрабатываются ЮKassa напрямую)

4. ХРАНЕНИЕ ДАННЫХ
- Сообщения хранятся 90 дней с момента отправки, затем удаляются автоматически
- При удалении аккаунта все данные удаляются безвозвратно в течение 30 дней

5. ПРАВА ПОЛЬЗОВАТЕЛЕЙ
Вы вправе:
- Запросить удаление аккаунта и всех данных (Настройки → Удалить аккаунт)
- Экспортировать свою переписку (функция в разработке)

6. БЕЗОПАСНОСТЬ
Мы применяем: TLS 1.3 для транспорта, E2E-шифрование (P-256 + AES-256-GCM) для сообщений, Argon2id для хранения PIN, Certificate Pinning в приложении.

7. КОНТАКТЫ
По вопросам конфиденциальности: support@iromashka.ru
Сайт: https://iromashka.ru
```
